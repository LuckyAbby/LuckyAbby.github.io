css 浮动以及常见的清除浮动

CSS 浮动

浮动很类似于印刷排版中的文字环绕效果。浮动的盒子可以向左或者向右移动，直到它的外边缘碰到包含盒子或另一个浮动盒子的边为止。由于浮动盒子不在文档的普通流中，相当于不占用空间，所以文档的普通流中的块框表现得就像浮动框不存在一样。

同时这种特性会导致包含浮动盒子的父元素塌陷，无法自动延伸高度，从而使得父元素的高度为 0 。

当我们将第一个 div 向左边浮动的时候，脱离正常流之后就会导致一个问题，使得 第二、三个 div 感知不到第一个 div 的存在，因此上移，这时候使得这个 div 盖住了第二个 div。

想解决这个问题，只要让三个 div 都浮动即可，但是这样会导致父容器的高度无法自动撑起来，这种问题后面来说怎么解决。

浮动的时候如果父元素的宽度不够，第三个 div 会自动向下浮动，直到有足够的宽度，如果浮动的时候发现高度不一致就会出现下面右边被卡住的情况。

清除浮动

上面也说了，浮动会导致一些问题出现，那么怎么清理浮动呢？

overflow

给浮动的父元素添加一个 overflow 属性，设置成 auto 或者 hidden 就会使得父容器的高度撑起来。

使用伪类元素

原理是在父元素上面使用 ::after 伪类元素，使用伪类元素可以在父元素后面创造隐藏的元素，将元素的 display 设置成block 或者 table-cell。

需要注意的是IE6，IE7不支持 ::after 伪元素，因此要添加 .clearfix{zoom:1} ；来触发haslayout。

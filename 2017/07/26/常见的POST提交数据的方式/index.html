常见的POST提交数据的方式

请求分为状态行、请求头以及消息主体，我们通过 POST 发送到数据是放在消息主体中的，通过请求头的 content-type 就能控制数据的提交方式，服务器端根据这个字段就能知道请求的消息主体是何种编码方式了，这样就能正确地对主体进行解析。

content-type 一般常用的有下面几种：

application/x-www-form-urlencoded

最常见的数据提交方式，也是表单的提交方式，对于原生表单如果不设置 enctype 属性，就是以这种方式提交数据。使用 类似 AJAX 库的时候默认的编码方式也是这个。这种方式提交的数据请求的数据被编码成 key=value&&key=value 的格式,类似于下图

application/json

这种提交方式主要是告诉服务器端传过来的数据是序列之后的 json 字符串的。这种方式也使用得十分平常。浏览器端传输数据的时候只要使用 JSON.stringify 就可以将一个 js 值转化成 JSON 对象。

json 格式对于复杂的数据结构处理比较方便，在浏览器里面能以十分直观的树形结构展示出来，比较适合做 RESTful api。

multipart/form-data

当需要在表单中上传文件的时候一般就需要这种数据格式。通常要设置表单enctype=”multipart/form-data”。同时我们需要选择一段数据作为“分割边界”（boundary属性），这个“边界数据”不能在内容其他地方出现，一般这个数据都是浏览器随机生成的。

选择了这个边界之后，浏览器便把它放在 Content-Type 里面传递给服务器，服务器根据此边界解析数据。下面的数据便根据boundary划分段，--用于区分数据快，后面的数据 ——WebKitFormBoundaryOGkWPJsSaJCPWjZP
就是标示区分包作用，每一段便是一项数据。
